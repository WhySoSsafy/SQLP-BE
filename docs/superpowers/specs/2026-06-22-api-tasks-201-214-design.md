# API 태스크 201~214 작업 설계

## 개요

SQLP 백엔드 API를 프론트엔드(현재 React → Vue 전환 예정)와 연동하기 위한
문서화 및 검증 작업 14개 티켓의 실행 설계.

**원칙:**
- 백엔드(`SQLP-BE`)만 수정. 프론트엔드(`SQLP-FE`) 미접촉.
- 티켓마다 지정된 브랜치 이름 사용.
- 작업 순서: 201 → 214 (번호 순 = 1순위 → 3순위).

---

## Git 브랜치 전략

- 기준 브랜치: `main` (현재 클린 상태)
- 각 티켓: `main`에서 지정 브랜치 생성 → 작업 → 커밋 → `main` 복귀
- 브랜치는 머지하지 않고 각자 커밋 유지 (PR 흐름)

---

## 티켓별 작업 명세

### docs/* 브랜치 — api.md 업데이트

각 브랜치에서 `api.md`에 해당 엔드포인트의 실제 응답 형식, 에러 케이스,
헤더 형식 등을 추가/보완해서 커밋.

| 브랜치 | 티켓 | 작업 내용 |
|--------|------|-----------|
| `docs/201/login-response` | 201 | POST /api/auth/login/ 성공/실패 응답 필드, 상태코드 정리 |
| `docs/202/signup-response` | 202 | POST /api/auth/register/ 성공/실패 응답 필드, 에러 메시지 형식 |
| `docs/204/auth-header` | 204 | 인증 필요 API의 Authorization 헤더 형식 (`Bearer <token>`) 명시 |
| `docs/205/analysis-validate-api` | 205 | POST /api/analysis/validate/ 요청 body, 성공/에러 응답 정리 |
| `docs/206/session-post-api` | 206 | POST /api/sessions/ 요청 body, 성공/중복 응답 정리 |
| `docs/208/api-spec` | 208 | `docs/api-spec.md` 신규 생성 — 프론트 연동용 통합 명세 |
| `docs/212/understanding-choices` | 212 | 이해도 choices 확정: `이해 / 애매 / 모름` (모델 현행 기준) |
| `docs/213/json-upload-policy` | 213 | JSON 업로드 정책 확정: 파일 아닌 JSON body 직접 수신 명시 |

### fix/* 브랜치 — 코드 검토 → 수정 or 검증 기록

코드를 검토해서 문제가 있으면 수정, 없으면 검증 결과를 api.md에 기록 후 커밋.

| 브랜치 | 티켓 | 예상 작업 |
|--------|------|-----------|
| `fix/203/jwt-token-issue` | 203 | LoginView가 access+refresh+user 정상 발급 확인 → 검증 기록 |
| `fix/207/session-list-api` | 207 | GET /api/sessions/ 필터, 응답 구조 확인 → 검증 기록 |
| `fix/209/calendar-api` | 209 | GET /api/calendar/ 실제 DB 기반 응답 확인 → 검증 기록 |
| `fix/210/study-comparison-api` | 210 | GET /api/study-comparison/ 응답 구조 확인 → 검증 기록 |
| `fix/211/concept-api` | 211 | GET /api/concepts/{id}/ 응답에 `person` 필드 불일치 → **코드 또는 스펙 수정** |
| `fix/214/growth-report-api` | 214 | `period` 파라미터가 날짜 필터 없이 echo만 됨 → api.md에 동작 명시 |

## 특이사항 (사전 파악된 이슈)

### 211 — 개념 상세 응답 person 필드 불일치
- `ConceptDetailView`의 `relatedProblems` 응답에 `person` 필드가 있음
- 기존 `api.md` 스펙에는 `person` 필드 없음
- fix/211 브랜치에서 api.md 스펙에 `person` 추가하는 방향으로 해결

### 214 — 성장 리포트 period 파라미터 미구현
- `period` 값을 받아서 응답에 echo는 하지만 실제 날짜 필터링 없음
- 전체 기간 데이터를 항상 반환하는 현재 동작을 api.md에 명시
