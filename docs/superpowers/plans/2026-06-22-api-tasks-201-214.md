# API 태스크 201~214 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** SQLP 백엔드 API 14개 티켓을 각자의 git 브랜치에서 문서화/검증/수정하여 프론트엔드 연동 준비를 완료한다.

**Architecture:** 각 티켓마다 `main`에서 지정 브랜치를 생성해 작업 후 커밋하고 `main`으로 복귀. docs/* 브랜치는 `api.md` 업데이트(208만 `docs/api-spec.md` 신규), fix/* 브랜치는 코드 수정 또는 검증 결과를 `api.md`에 기록.

**Tech Stack:** Django 4.x, DRF, djangorestframework-simplejwt, PostgreSQL, pytest-django

## Global Constraints

- 백엔드(`SQLP-BE`) 디렉토리만 수정. `SQLP-FE`는 절대 건드리지 않는다.
- 모든 작업은 `main` 브랜치를 기준으로 각 티켓 브랜치에서 진행한다.
- JWT: access token 유효 1시간, refresh token 유효 7일 (settings.py 기준).
- understanding choices 값: `이해` / `애매` / `모름` (모델 정의 기준).
- 공통 에러 응답 형식: `{"ok": false, "code": "ERROR_CODE", "message": "...", "errors": [...]}`
